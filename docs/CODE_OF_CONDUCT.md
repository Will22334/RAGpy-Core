Code of Conduct



Purpose

The purpose of this Code of Conduct is to ensure that RAGpy remains a welcoming, respectful, and productive environment for all contributors. This document outlines expected behavior, unacceptable behavior, and the process for reporting issues. All participants are expected to uphold these standards in all project spaces, including GitHub issues, pull requests, discussions, documentation, and any related communication channels.



Expected Behavior

Contributors, maintainers, and users are expected to demonstrate respect and professionalism, use inclusive and welcoming language, provide constructive feedback, collaborate with empathy, focus on what is best for the project, and follow project guidelines including coding standards and contribution processes.



Unacceptable Behavior

Unacceptable behavior includes harassment, intimidation, discrimination, offensive or derogatory comments, personal attacks, disruptive behavior, publishing private information without permission, excessive or aggressive criticism without constructive intent, and any behavior that violates applicable laws or regulations.



Responsibilities of Maintainers

Project maintainers are responsible for enforcing this Code of Conduct, reviewing and addressing reported violations, taking appropriate corrective action, and ensuring that all contributors feel safe and respected. Maintainers may remove, edit, or reject contributions that do not align with this Code of Conduct or the project’s goals.



Reporting Violations

Violations may be reported to the project maintainers. Reports should include a description of the incident, where it occurred, and any relevant context or evidence. All reports will be handled confidentially. Retaliation against individuals who report violations is prohibited.



Enforcement

Violations may result in warnings, removal of comments or contributions, temporary suspension, or permanent removal from the project community. Enforcement actions will be determined by project maintainers based on the nature of the violation.



Scope

This Code of Conduct applies to all project spaces including GitHub repositories, issue trackers, pull requests, discussions, documentation, and any other communication channels associated with RAGpy. It also applies when individuals represent the project in public or private settings.



Acknowledgment

This Code of Conduct is adapted from widely used open-source community standards, including the Contributor Covenant, to ensure clarity, fairness, and consistency.
