Contributing Guidelines



Thank you for your interest in contributing to RAGpy. This project welcomes contributions from the community and aims to maintain a clear, consistent, and high‑quality codebase. These guidelines describe how to contribute effectively and responsibly.



Ways to Contribute

You can contribute by reporting bugs, suggesting enhancements, improving documentation, submitting code changes, or adding new features. All contributions are reviewed to ensure they align with the project’s goals and standards.



Reporting Issues

Use the GitHub issue tracker to report bugs or request features. Provide a clear description, steps to reproduce the issue, expected behavior, and any relevant logs or screenshots. Well‑written issues help maintainers address problems more quickly.



Submitting Pull Requests

Before submitting a pull request, ensure your changes are focused, well‑documented, and tested. Follow these steps:



Fork the repository and create a new branch for your changes.



Make your changes in small, logical commits.



Write clear commit messages describing what you changed and why.



Ensure your code follows the project’s style and structure.



Add or update tests when appropriate.



Update documentation if your changes affect usage or behavior.



Open a pull request describing your changes and referencing any related issues.



Code Style

Follow consistent naming conventions, formatting, and structure throughout the project. Write clear, readable code with comments where necessary. Avoid unnecessary complexity and ensure functions and modules have a single, clear purpose.



Testing

All new features and bug fixes should include tests. Ensure existing tests pass before submitting a pull request. Tests help maintain reliability and prevent regressions.



Documentation

Documentation is essential for usability. Update or add documentation when introducing new features, changing behavior, or fixing issues that affect usage. Clear documentation helps users and contributors understand how the project works.



Communication

Be respectful and constructive when interacting with other contributors. Collaboration and clear communication help maintain a positive and productive environment.



Licensing

By contributing to RAGpy, you agree that your contributions will be licensed under the same license as the project. Ensure you have the right to submit any code or content you contribute.



Review Process

Maintainers review all pull requests. Feedback may be provided, and changes may be requested before merging. Maintainers may close pull requests that are out of scope, incomplete, or inconsistent with project goals.



Acknowledgment

Thank you for helping improve RAGpy. Your contributions support the growth and reliability of the project and help the community build better RAG systems.
